## 第零部分：报告生成前置条件（v3.0 硬约束）

> 生成选股报告前必须满足以下条件，**否则必须拒绝生成正式报告**
> ⚠️ **v3.0 重大升级：本部分所有规则均为硬约束，脚本必须 assert 实现，违反则报告生成失败**

**【硬约束检查项 —— 任一不满足则 assert 终止报告生成】**

1. **版本一致性（v3.0新增）**：报告生成脚本必须在启动时读取 `docs/stock.md` 头部版本号（如 v3.0），与脚本内嵌版本号比对。不一致时**必须**输出警告并拒绝生成报告，或自动同步后继续。
   ```python
   # 示例实现
   with open(f"{PROJECT_DIR}/docs/stock.md") as f:
       import re
       version = re.search(r'版本[:：v\s]+v?(\d+\.\d+)', f.read())
       assert version.group(1) == SCRIPT_VERSION, f"版本不一致: script={SCRIPT_VERSION}, stock.md={version.group(1)}"
   ```

2. **交易日校验**：今日必须是交易日。周六/周日/法定假日 → 输出"⚠️ 休市日报告，仅供参考，仓位建议：0%，禁止买卖"，然后终止后续选股流程。

3. **数据完整性（v3.0硬约束升级）**：上证指数、深证成指、创业板指、科创50 **必须全部获取成功**。任一指数获取失败时：
   - **必须**执行 `assert False, "数据不完整，禁止生成报告"`
   - 输出格式化的数据状态声明（哪些指数获取成功/失败）
   - **强制**将仓位建议设为 `0%`，标注 `"⚠️ 数据不可用，禁止开仓"`
   - 禁止在数据缺失时使用昨日数据 fallback 伪装成"今日数据"

4. **数据时间戳校验（v3.0新增）**：获取的指数数据必须包含有效时间戳，且时间戳日期必须为今日。不接受昨日快照数据作为"今日数据"。

5. **成交额校验**：大盘成交额数据必须可用。若获取失败 → 同数据完整性处理。

**【数据缺失时的硬性输出格式（v3.0新增）】**
若上述检查未通过，脚本必须输出以下内容并终止：
```
⚠️ 今日数据不可用，请勿执行任何操作
数据状态：上证指数❌ / 深证成指❌ / 创业板指❌ / 科创50❌
仓位建议：0%（强制，禁止开仓）
版本状态：未验证（stock.md vX.X ≠ 报告版本 vX.X）
```

## 第一部分：大盘环境判断（决定是否开仓）

> 版本：v2.8（复盘优化：增加市场广度辅助判断）

### 规则

根据当日大盘指数涨跌幅和成交额，决定仓位上限：

**【市场广度辅助判断 - v2.8新增】**
当上涨股票数 < 全市场 40%（市场广度差）时，在以下仓位建议基础上再降一档：
即使指数微涨，市场广度差说明真实趋势向下，应谨慎。

| 大盘条件 | 模式 | 仓位建议 |
|----------|------|----------|
| 涨跌幅 > +1.5% AND 成交额 > 2万亿 | 积极模式 | 5-7成仓，可追击强势股 |
| 涨跌幅 -1% ~ +1.5% | 中性模式 | 3-5成仓，谨慎选股 |
| 涨跌幅 < -1% | 防守模式 | ≤3成仓，或空仓观望 |
| 科创50跌幅 > 1.5%（v2.7下调） | 警报模式 | 空仓为主，禁止追买 |
| 市场广度 < 40%（v2.8新增） | 降一档 | 在上述仓位基础上减一档 |

### 判断顺序

1. 读取当日实时指数（上证/深证/创业板/科创50）
2. 判断各指数涨跌幅和成交额
3. 取最弱的指数作为仓位依据
4. 成交额以是否超过2万亿为分水岭

## 第二部分：板块动量筛选（缩小选股范围）

### 规则

- 统计各行业板块近5日涨跌幅排行
- 只选**近5日动量排名靠前的前3板块**
- 排除**近5日已大涨超过15%的过热板块**（防止追高）

### 龙头股强制纳入规则（v2.8新增）

**【强制要求】每个优先板块内，必须将近5日涨幅最大的前3只股票纳入候选池**
- 即使估值偏高、即使股价位置较高，也必须纳入
- 防止选股池只捡小票、错过龙头主导行情
- 龙头股即使高位，评分时应使用"趋势确认"逻辑（+5分），而非"追高风险"逻辑（-3分）

### 优先板块条件

```
近5日涨幅 0%~10%（有动量但不过热）
近20日涨幅 < 30%（未进入主升浪）
```

### 禁止追入的情况

- 板块近5日涨幅 > 15%（RSI过度超买）
- 板块当日成交量创历史天量（可能是出货）
- 板块K线出现高位吊颈线/黄昏之星

### 动态选股池（v3.0新增）

> ⚠️ **重要警告：** 选股池**不得**依赖固定预设股票列表。fetch_stock_data.py 中的 `focus_stocks`（14只预设股票）是观察列表，不是选股池。

**选股池必须来自以下三个渠道（按优先级）：**

1. **板块动量扫描（主要来源）：** 调用 eastmoney_scraper.py 获取各行业板块近5日涨跌幅排行，从前3强势板块中各取涨幅前5只个股
2. **龙头股强制纳入（强制要求）：** 每个优先板块内近5日涨幅最大的前3只股票**必须**纳入候选池，无论估值高低
3. **用户自定义关注（补充来源）：** focus_stocks 作为补充参考，不得作为主要候选池

**脚本实现要求（v3.0）：**
```python
# ❌ 错误做法：只用 focus_stocks 14只股票
candidates = focus_stocks  # 14只固定股票

# ✅ 正确做法：全市场板块扫描
sector_leaders = eastmoney_scraper.get_sector_momentum_top()  # 前3板块各5只
all_candidates = sector_leaders + get_forced_leader_stocks()  # 龙头强制纳入
```

**禁止事项：**
- 禁止将昨日收盘价数据伪装成"今日实时数据"
- 禁止在数据获取失败时使用 focus_stocks 快照数据生成完整报告

## 第三部分：个股精准筛选（17条指标）

> 以下17条指标，筛选时需**至少满足其中一条**，优先选择**同时满足多条**的个股。

### 指标1：低估值稳健型
```
条件：市销率 0~5倍；成交额 5000万~5亿；毛利率同比 ≥ -5%；沪深主板；上市时间 > 1年
```

### 指标2：MACD金叉反弹型
```
条件：连续上涨 1~2天；股价 1~10元；换手率 3%~8%；日线MACD金叉；OBV 5日变化率 > 0
```

### 指标3：放量突破型
```
条件：当日成交量 > 前日1.5倍；股价 < 200元；当阳线；收盘价 > 10日均线；
      当日最高价创50日新高；创业板非ST
```

### 指标4：量价齐升型
```
条件：涨幅 3%~6%；总市值 60亿~300亿；量比 2~5；换手率 6%~15%；主板
```

### 指标5：均线多头稳健型
```
条件：5日均线 > 10日均线 > 20日均线；近5天最大跌幅 < 7%；
      1个月内有涨停板；成交量 > 5日均量；流通市值 < 100亿；
      收盘价 > 20日均线；收盘价 < 20元
```

### 指标6：均线突破型
```
条件：当日涨幅 > 5%；非北交所；板块为创业板或科创板；
      当日收盘价 > 10日均线；前一日收盘价 < 10日均线；换手率 > 5%
```

### 指标7：量能健康型 ⭐
```
条件：成交量 100万~5000万股；换手率 3%~20%；股价 5~20元；5日均线向上
```

### 指标8：涨停基因型
```
条件：近50日涨停 1~7次；5日均线与10日均线多头排列；主板；
      近1年区间涨幅 ≤ 100%；收盘价 > 60日均线；非ST
```

### 指标9：中小盘动量型
```
条件：总市值 50亿~200亿；换手率 5%~10%；涨幅 3%~5%；量比 1~2
```

### 指标10：缩量整理型
```
条件：成交量较前一日缩小；连续5日收盘价 > 10日均线；
      非ST非科创板非北交所非创业板；当日涨幅 < 前一日；近两日未触及涨停
```

### 指标11：资金流入型 ⭐
```
条件：5日大单净流入 > 0；市盈率 10~1000倍；全部A股；5日涨幅 0%~5%；量比 1.2~1.5
```

### 指标12：低位放量型
```
条件：总市值 < 400亿；近7天涨跌幅由小到大排名（低位优先）；
      成交量 > 10日均量；股价上穿30日均线；10日均线上穿20日均线；
      市盈率处于历史中位数以下；收盘价 > 3元
```

### 指标13：业绩增长型 ⭐
```
条件：非退市非ST；涨幅 3%~5%；净利润同比增长率 > 0；量比 > 1.5；成交额 > 10亿
```

### 指标14：超跌反弹型
```
条件：流通市值 50亿~1500亿；量比 1~5；跌幅 5%~10%；成交额 8亿~150亿；沪深主板
```

### 指标15：小盘突破型
```
条件：流通市值 < 50亿；换手率 1%~5%；收盘价上穿20日均线；振幅 < 5%；沪市或深市
```

### 指标16：温和放量型
```
条件：量比 > 2；当成交量创近5日新高；涨幅 5%~10%；K线收阳且未涨停；
      上影线长度 > 实体长度（仙人指路形态）
```

### 指标17：业绩超跌型
```
条件：营业收入 ≥ 20亿；归属母公司净利润 > 0；跌幅 ≤ -3%（超跌反弹预期）
```

### 指标18：主力净流入确认型 ⭐（v2.1新增）
```
条件：
  - 5日大单净流入 > 0（确认资金持续入场）
  - 成交额5亿~100亿（避免超大盘股或迷你盘）
  - 当日涨幅 2%~6%（有利润空间但不追高）
  - 收盘价在当日区间中位以上（拒绝冲高回落仙人指路）
  - 股价不在历史高位（近20日涨幅 < 30%）
```

## 第四部分：技术指标库（辅助判断）

### 趋势确认信号

| 信号 | 类型 | 含义 |
|------|------|------|
| MACD 金叉 | 买入 | 5日均线上穿20日均线 |
| MACD 死叉 | 卖出 | 5日均线下穿20日均线 |
| 收盘价 > 20日均线 | 买入参考 | 短期趋势向上 |
| 收盘价 < 20日均线 | 卖出参考 | 短期趋势向下 |

### 动量信号

| 信号 | 条件 | 含义 |
|------|------|------|
| 近5日涨幅 | 0%~10% | 有动量但未过热 |
| 近20日涨幅 | < 30% | 未进入主升浪，安全边际 |
| 量比 | > 1.5 | 资金关注度高 |
| 换手率 | 3%~15% | 活跃度适中，不过分换手 |

### 风险信号（满足任一则过滤）

| 信号 | 说明 |
|------|------|
| KDJ 高位死叉（J值 > 90） | 短期超买，可能回调 |
| 收盘价 < 20日均线 | 短期趋势破位 |
| 近5日最大单日跌幅 > 5% | 波动过大，风险高 |
| 上影线过长（> 实体2倍） | 冲高回落概率大 |
| 高位放量大阴线 | 主力出货嫌疑 |
| **开盘涨幅 > 2.5%** | 高开过多，尾盘回落概率大（v2.1新增） |
| **开盘价 > 3日内最高价** | 价格已透支空间（v2.1新增） |
| **成交量创近期天量但价格不涨** | 主力借利好出货嫌疑（v2.1新增） |
| **收盘价 < 开盘价 - 1.5%** | 高开低走仙人指路，应过滤（v2.2新增） |
| **近5日涨幅 > 15%** | 即使当日回调也不推荐，高位补跌风险大（v2.2新增） |
| **成交额创近20日天量且涨幅 < 2%** | 主力借利好出货嫌疑，应降权或过滤（v2.2新增） |
| **（v2.7新增）** | **禁止追高规则** |
| 开盘涨幅 > 3% 且收盘涨幅 < 开盘涨幅 - 1% | 高开低走，禁止推荐 |
| 近3日累计涨幅 > 15% | 高位补跌风险大，禁止推荐 |

## 第五部分：选股流程（执行顺序）

### Step 1：大盘环境判断
```
读取当日大盘数据（指数涨跌幅 + 成交额）
↓
判断模式（积极/中性/防守/警报）
↓
决定仓位上限（7成/5成/3成/空仓）
↓
若警报模式 → 停止选股，直接空仓
```

### Step 2：板块动量筛选
```
搜索近5日板块涨跌幅排行
↓
取前3强势板块
↓
过滤近5日涨幅 > 15%的过热板块
↓
确定优先关注板块（最多3个）
```

### Step 3：个股指标筛选 + 追高过滤（v3.0强制）
```
获取优先板块内全部个股
↓
逐条应用17条指标
↓
标记同时满足多条指标的个股（优先度高）
↓
【v3.0新增】应用追高过滤（apply_chase_filter）
  → removed：今日>7%且近5日>10%且20日新高 → 从所有列表中移除
  → demoted：今日>5%且无回调信号 → 降权至最末
  → passed：通过过滤，进入推荐列表
↓
验证：推荐列表 ∩ 风险提示 = ∅（必须互斥）
↓
按指标满足数量排序，取前5~10只
```

### Step 4：精选与排序
```
综合指标满足数量 + 当日涨跌幅 + 成交额
↓
给出推荐个股及优先顺序
↓
每只股票注明：
  - 符合的指标编号
  - 买入价格区间（参考当日涨停价 - 3%以内）
  - 止损位（买入价 - 5%）
  - 目标位（买入价 + 10%~15%）
  - 持仓时长建议（3~5个交易日）
```

### Step 4.5：收盘前30分钟确认（v2.1新增）
```
收盘前30分钟扫描候选股：
↓
若股价高于开盘价（拒绝高开低走）→ 保留候选
若股价低于开盘价且跌幅 > 1% → 从候选中剔除或降权
↓
若成交量在合理范围（非天量出货）→ 保留
若成交额创近期天量但涨幅 < 1% → 降权或剔除
↓
最终确认当日买入清单
```

### Step 5：操作执行
```
根据仓位上限，按优先顺序分配仓位
单只股票仓位不超过总仓位30%
止损单设置好后再挂买单
```

## 第六部分：技术分计算规则（v2.6修正）

| 情形 | 技术加分 | 说明 |
|:----:|:--------:|:-----|
| 回调-3%~0%，且近5日涨幅 < 10% | +10分 | 最佳买点区，安全垫好 |
| 回调-7%~-3%，近5日涨幅 < 5% | +15分 | 超跌反弹，安全边际高 |
| 回调-7%~-3%，近5日涨幅 > 10% | +3分 | 高位接刀，风险大于机会 |
| 当日涨幅 < -2% | -10分 | 下跌趋势，风险大 |
| 当日涨幅 > 5% 且近5日涨幅 < 15% | +5分 | 动量确认，积极加成 |
| 温和上涨0~5% | +10分 | 趋势健康 |
| 大幅上涨>5%，近5日涨幅 < 10% | +8分 | 趋势启动初期，积极确认（v2.7上调） |
| 大幅上涨>5%，近5日涨幅 10%~20% | +2分 | 趋势确认，但已累积涨幅（v2.7下调） |
| 大幅上涨>5%，近5日涨幅 > 20% | -8分 | 高位追涨，风险大于机会（v2.7下调） |
| **强势股识别加分（v2.6新增，v2.8修订）** | | |
| 今日涨幅 3%~7%，近5日涨幅 < 15% | +5分 | 趋势确认，动量健康 |
| 今日涨幅 7%~10%，近5日涨幅 < 15% | **-3分（v2.8修订）** | 强势但追高风险大，应降权而非加分 |
| 今日涨幅 7%~10%，近5日涨幅 < 15%，且为龙头股强制纳入 | +5分（v2.8新增） | 龙头股享受例外，即使追高也有板块支撑 |
| 涨停>9%，近5日累计涨幅 < 15% | +10分 | 健康涨停，看涨延续（v2.5保留） |
| 涨停>9%，近5日累计涨幅 15%~25% | 0分 | 中性，不加分不降权 |
| 涨停>9%，近5日累计涨幅 > 25% | -10分 | 连续涨停过热，风险大 |

> ⚠️ **重要原则：** "大幅回调"本身不是买入信号，需结合近5日涨幅综合判断。若近5日已大涨10%以上，则当日继续下跌可能是主力出货后的补跌，不应视为超跌反弹机会。
>
> **v2.4/v2.6修订核心逻辑：** 涨幅大不是问题，涨在高处才是问题。趋势刚启动的强势股（如近5日涨幅<15%的放量大涨）应获得正常加分或额外加分，而非惩罚。
>
> **颜色标注规范（v2.6新增）：** 🔴红色=下跌/负面，🟢绿色=上涨/正面，⚠️黄色=中性/警告。避免"🟢下跌"等矛盾表述。

## 第七部分：禁止操作规则（v3.0 强制执行）

> ⚠️ **v3.0 升级：** 以下所有规则必须在脚本层面通过代码 assert/filter 实现，而非仅在文档中描述。推荐列表的生成过程必须可审计——每只股票的纳入/移除/降权必须有明确的代码日志。

**【硬约束 —— 脚本必须实现，否则报告无效】**

1. **禁止在警报模式下追买任何股票** — 警报模式下选股流程必须完全终止
2. **禁止买入ST股、退市股** — filter 层必须排除
3. **禁止追买近5日涨幅超过15%的股票** — filter 层必须排除
4. **单只股票仓位不得超过总仓位的30%** — 分配逻辑层控制
5. **亏损超过5%必须止损，不得补仓** — 复盘追踪逻辑
6. **持仓时间不得超过10个交易日** — 持仓追踪逻辑
7. **当日涨幅 < -2% 的股票，谨慎推荐，需明确下跌原因**

**【追高过滤的代码实现要求（v3.0强制）】**

```python
def apply_chase_filter(candidates, stock_data):
    """
    v3.0 强制执行：追高过滤
    返回: (passed_candidates, demoted_candidates, removed_candidates)
    """
    passed, demoted, removed = [], [], []
    for stock in candidates:
        today_pct = stock.get('change_pct', 0)
        near5_pct = stock.get('near5_change_pct', 0)
        is_20d_high = stock.get('price_near_20d_high', False)
        has_pullback = stock.get('has_pullback_signal', False)  # 回调-3%~0%

        # 规则1：同时满足 → 自动移除（不出现在任何推荐列表）
        if today_pct > 7 and near5_pct > 10 and is_20d_high:
            removed.append(stock)
            continue

        # 规则2：满足以下 → 降权至最低优先级
        if today_pct > 5 and not has_pullback:
            stock['priority'] = 'low'
            demoted.append(stock)
            continue

        passed.append(stock)

    return passed, demoted, removed
```

**【关键约束：推荐列表与风险提示必须互斥（v3.0新增）】**
```python
# ✅ 正确实现
recommended = [s for s in candidates if s not in removed and s.get('priority') != 'low']
warning_list = removed + [s for s in demoted if s['change_pct'] > 5]
assert set(recommended).isdisjoint(set(warning_list)), "推荐列表与风险提示不得重叠"
```

**【禁止高开低走类股票（v3.0强制）】**：
- 开盘涨幅>5%，收盘涨幅<开盘涨幅-1.5%
- 近3日出现连续高开低走（每日开盘涨幅>2%）
- 成交额创近20日天量但涨幅<1%（主力出货嫌疑）

**【禁止追高规则细化】**：
- 推荐个股应优先选择回调股（跌幅-3%~0%为最佳买点区）
- 强势股加分条件：今日涨幅>7%且近5日涨幅<15% → -3分，追高风险大
- 回调-3%~0%且近5日涨幅<10% → +15分，最佳买点区

> ⚠️ **v2.9/v3.0 的关键区别：** v2.9只描述了"追高危险股应移除"的意图，v3.0要求必须通过代码 `assert set(recommended).isdisjoint(set(removed))` 确保推荐列表与风险提示完全互斥。
