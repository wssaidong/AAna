## 七、核心问题分析

### 问题1：大盘数据获取是致命漏洞（Critical）
- 报告在无实时数据情况下强行生成，导致所有判断建立在"数据待获取"的基础上
- 仓位建议、市场情绪判断完全失效
- **根因**：脚本设计缺陷——报告生成时若数据获取失败，应该：
  1. 拒绝生成报告，或
  2. 明确标注"数据不可用，本报告仅供参考，勿执行"
  3. 自动回退到"防守模式"（≤30%仓位）

### 问题2：选股池严重偏离真实强势方向（Major）
- 海光信息（+8.2%，AI算力最强标的）完全未被推荐
- 寒武纪（+2.2%）、中微公司（+2.6%）等半导体龙头均未纳入
- 推荐池几乎全是创业板小盘股（瑞晨环保、普瑞眼科等），4-24当天这些板块整体偏弱（创业板-1.41%）
- **根因**：选股指标体系偏向小盘/技术面，忽略了"板块动量"和"龙头股"筛选

### 问题3：技术分计算规则自相矛盾（Major）
- v2.8新增规则："强势股识别：今日涨幅>7%且近5日涨幅<15% → -3分（原+8分）"——**但报告版本是v2.3，应用的是+8分逻辑**
- 实际结果显示：这些"强势股"被加+8分后评分达67，却没有任何基本面或板块逻辑支撑
- **根因**：版本管理混乱，v2.8的新规则未同步到实际生成脚本

### 问题4：推荐逻辑反常识（Medium）
- 在大盘整体偏弱（3/4指数下跌）的日子，推荐了38只全是"今日已大涨5-9%"的股票
- 这意味着在非强势市场中追高，明显违背"最佳买点"原则
- **根因**：选股时未引入"大盘市场广度"过滤——若全市场上涨股票<40%，应自动降权追高型股票

## 八、优化建议（stock.md修订方案）

### 修订1：增加「报告生成数据校验」机制（Critical）

在 stock.md 第五部分选股流程中，**Step 1 之前**增加前置检查：

```
【选股报告生成前置条件】
1. 检查当日是否为交易日（周六/周日/法定假日 → 禁止生成或标注"休市日"）
2. 检查大盘指数数据获取是否成功（任一指数失败 → 拒绝生成报告）
3. 若指数获取部分失败（如仅能获取上证），以最弱指数作为仓位依据
4. 若所有数据获取失败 → 输出"今日数据不可用，请手动检查"，不生成正式报告
```

### 修订2：大盘广度过滤（Major）

在第一部分大盘环境判断中，增加「市场广度」辅助判断：

```
当以下条件全部满足时，在原有仓位建议基础上再降一档：
  - 上涨股票数 < 全市场40%（市场广度差）
  - 即使大盘指数微涨，也应谨慎
```

### 修订3：龙头股强制纳入规则（Major）

在第二部分板块动量筛选中，增加：

```
【龙头股优先规则】
板块内近5日涨幅最大的前3只股票，即使估值偏高，也必须纳入候选池
（防止选股池只捡小票、错过龙头）
```

### 修订4：版本锁定（Medium）

- 当前 stock.md 显示版本 v2.7，但脚本生成报告为 v2.3，说明版本已脱节
- **建议**：在 generate_report.py 脚本开头加入版本读取逻辑，每次生成时自动同步 stock.md 版本号

### 修订5：技术分计算一致性（Medium）

删除 stock.md 第六部分中相互冲突的规则，合并为：

```
强势股（近5日涨幅<15%）当日涨幅 7%~10%：+5分（原+8分降为+5分，减少追高激励）
理由：即使趋势健康，过快涨幅也会在次日面临短线获利了结压力
```
