## 六、核心问题分析

### 问题1：数据获取脚本执行失败
- **现象：** 报告生成时显示"数据待获取"，所有大盘指标均为空
- **根因：** 选股脚本调用 fetch_stock_data.py 时未成功获取实时数据
- **影响：** 导致大盘判断基于错误前提，若当日大盘大跌将造成严重亏损

### 问题2：技术分计算规则与实际数据脱节
- **现象：** 五洲新春-0.55%却获得技术分78（高分），而科大国创-3.17%未被推荐
- **根因：** 技术分计算主要依赖"符合指标数量"而非当日涨跌幅趋势
- **影响：** 可能推荐弱势股，错失强势股

### 问题3：科大国创（300352）大跌3.17%未被标记风险
- **现象：** 科大国创今日跌幅达3.17%，但未被列入任何预警名单
- **根因：** 该股不在报告关注的热点板块内，缺乏主动扫描机制
- **影响：** 若用户持有将承受较大亏损

## 七、优化建议

### 建议1：修复数据获取流程（紧急）
```
修改选股报告生成流程：
1. fetch_stock_data.py 必须在选股脚本之前执行
2. 数据获取失败时，报告应显示"数据异常，暂停推荐"而非"数据待获取"
3. 增加数据获取超时检测（>30秒则标记失败）
```

### 建议2：优化技术分计算（重要）
```
当前问题：技术分主要由指标符合数量决定，涨跌幅权重不足
建议调整：
- 当日涨幅 < -2% 的股票，技术分自动 -10分
- 当日涨幅 > 5% 且近5日涨幅 < 15%，技术分 +5分（动量加成）
- 避免推荐当日跌幅 > 2% 的股票（除非是超跌反弹逻辑）
```

### 建议3：增加全市场异动监控（重要）
```
建议增加异动监控逻辑：

- 全市场扫描当日涨幅 > 7% 的股票（机会发现）
- 不限于热点板块，提高覆盖面
```
