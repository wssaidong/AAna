## 六、核心问题分析

### 问题1：大盘数据获取失败（严重）

- **现象**：选股报告中上证/深证/创业板/科创50全部显示"数据待获取"
- **影响**：无法判断大盘环境，仓位建议无数据支撑
- **原因**：`fetch_stock_data.py` 在选股报告生成时未能成功写入数据到报告模板
- **建议**：检查 `generate_report.py` 中数据写入时机，确保数据获取完成后才生成报告

### 问题2：涨停股评分逻辑严重失真（严重）

- **现象**：
  - 新易盛(300502) 今日+9.58%（涨停级），综合评分仅50分，标注"⭐ 不推荐"
  - 三安光电(600703) 今日+9.98%（涨停），综合评分仅47分，标注"⭐ 不推荐"
- **问题**：根据 stock.md v2.5 规则：
  ```
  涨停>9%，且近5日累计涨幅<15% → +10分（健康涨停，看涨延续）
  ```
  新易盛近5日涨幅约9.58%，三安光电约9.98%，均应+10分，而非-15分
- **根因**：技术分计算时，未正确应用涨停加分规则，导致评分失真

### 问题3：仓位建议偏乐观

- **现象**：报告建议50-70%仓位
- **实际**：上证-0.10%（弱），创业板+1.43%（强），分化明显
- **建议**：分化市况下应降一档，30-50%更合适

### 问题4：新宏泽(002836) 数据缺失

- **现象**：复盘时无法找到新宏泽的实际数据
- **建议**：补全数据获取标的列表

## 七、优化建议

### 优先级1：修复涨停股评分失真（必须）

**当前 `docs/stock.md` 第六部分技术分计算：**

```
涨停>9%，且近5日累计涨幅<15% → +10分（健康涨停）
涨停>9%，近5日累计涨幅>20% → -10分（连续涨停过热）
```

**问题**：规则已正确，但执行层可能未匹配到对应股票。

**建议优化**：
1. 在技术分计算模块中，明确对"涨停>9%"的股票应用独立加分分支
2. 增加日志输出，标记每个候选股的加分来源（技术面/基本面/涨停加分）
3. 重新评估新易盛/三安光电：按规则应得70+10=80分左右（而非50/47分）

### 优先级2：修复大盘数据写入时机

**建议**：在 `generate_report.py` 的 `git_pull()` 之后，增加 `wait_for_data()` 逻辑，确保 stock_data.json 完整写入后再生成报告。

### 优先级3：仓位建议应考虑分化市况

**建议**：当上证/创业板涨跌幅差异>1%时，仓位降一档处理。
